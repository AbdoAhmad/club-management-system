📘 دليل دمج CKEditor 5 مع Livewire 3 (النسخة الاحترافية)

هذا الدليل يلخص أفضل ممارسة لدمج محرر النصوص CKEditor مع إطار العمل Livewire، بناءً على تجربة عملية لضمان الأداء العالي وسلامة البيانات.

1. الهيكل الأساسي (Blade Template)

يجب وضع المحرر داخل حاوية تحتوي على wire:ignore لمنع Livewire من إعادة بناء العنصر ومسح حالة المحرر.

<div class="mb-4" wire:ignore>
    <label>وصف اللاعب</label>
    <!-- textarea عادية بدون wire:model -->
    <textarea id="player_description" class="input-field">{!! $description !!}</textarea>
</div>

2. السر في الأداء: المزامنة اللحظية المؤجلة (Javascript)

بدلاً من المزامنة التقليدية التي تستهلك موارد السيرفر، نستخدم التحديث المؤجل (Deferred Updates).

ClassicEditor.create(document.querySelector('#player_description'), {
    // الإعدادات الخاصة بك...
}).then(editor => {
    // المزامنة مع كل حرف يكتبه المستخدم
    editor.model.document.on('change:data', () => {
        const data = editor.getData();
        
        // استخدام @this.set مع المعامل الثالث 'true'
        // هذا يحدّث الداتا في المتصفح فقط دون إرسال طلب للسيرفر فوراً
        @this.set('description', data, true);
    });
});

3. التعامل مع نظام التنقل (Livewire Navigated)

في تطبيقات Livewire 3، التنقل بين الصفحات لا يعيد تحميل الصفحة بالكامل. لذا يجب تنظيف الذاكرة وإعادة بناء المحرر.

document.addEventListener('livewire:navigated', async () => {
    // 1. مسح النسخ القديمة
    if (myEditorInstance) {
        await myEditorInstance.destroy();
        myEditorInstance = null;
    }
    // 2. إعادة التشغيل
    initCKEditor();
});

4. التنسيق الجمالي (Premium UI)

لجعل المحرر يبدو كجزء أصيل من نظامك، يجب تخصيص CSS ليتوافق مع باقي الحقول:

.ck-editor__editable, .ck-content, .ck-content * {
    color: #000000 !important;
    opacity: 1 !important;
}

.ck-editor.ck-focused {
    box-shadow: 0 0 0 3px rgba(34, 197, 94, 0.1) !important;
    border-color: #22c55e !important;
}
✅ لماذا هذا الحل هو الأفضل؟
  • سرعة فائقة في الكتابة.
  • استقرار كامل للبيانات (لا وجود للـ null).
  • توافق تام مع الـ SPA behavior الخاص بـ Livewire 3.
